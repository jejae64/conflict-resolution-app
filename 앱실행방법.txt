이렇게 에러 메시지를 그대로 붙여넣고 설명하면 됩니다:

app/api/analyze/route.ts에서 JSON.parse 에러가 반복적으로 발생하고 있어.

에러 종류:
1. Expected ',' or '}' after property value in JSON at position 2341
2. Bad control character in string literal in JSON at position 206  
3. Unterminated string in JSON at position 2196

route.ts 파일을 읽고 원인을 파악해서 다음을 수정해줘:
- max_tokens를 4096으로 늘리기
- stop_reason === "max_tokens" 체크 추가
- JSON 파싱 실패 시 문자열 내 줄바꿈만 이스케이프 후 재시도하는 방어적 파싱 로직 추가
- 제어문자 sanitize 추가
